**📊 Customer Shopping Behavior Analysis**

**🔍 Overview**
This project focuses on analyzing customer shopping behavior using a complete data analytics workflow. It covers data extraction, cleaning, exploratory data analysis (EDA), SQL-based querying, and visualization through an interactive dashboard. The goal is to derive meaningful insights that can help businesses make data-driven decisions.

**📁 Dataset**
The dataset contains customer transaction and purchasing details.
Includes attributes such as customer demographics, purchase history, product categories, and spending patterns.
Data was preprocessed to handle missing values, inconsistencies, and duplicates.

**🛠️ Tools & Technologies**
Python – Data loading, cleaning, and EDA
MySQL – Data storage and querying
Power BI – Dashboard creation and visualization
Gamma – Presentation (PPT) creation

**⚙️ Project Workflow**
1. Data Loading & Cleaning (Python)
Imported dataset using Pandas
Handled missing values and duplicates
Standardized and formatted columns
2. Exploratory Data Analysis (EDA)
Analyzed customer behavior patterns
Identified trends in spending and product categories
Used visualizations for better understanding
3. SQL Analysis (MySQL)
Created database and tables
Wrote queries to extract key business insights
Performed aggregations, joins, and filtering
4. Dashboard Creation (Power BI)
Built an interactive dashboard
Visualized KPIs like total sales, customer segments, and trends
Added filters and slicers for dynamic analysis
5. Reporting & Presentation
Compiled insights into a structured report
Created a professional presentation using Gamma

**📊 Dashboard Highlights**
Sales trends over time
Customer segmentation analysis
Top-performing products and categories
Key performance indicators (KPIs)

**📈 Results & Insights**
Identified high-value customer segments
Discovered top-selling product categories
Analyzed seasonal trends in customer purchases
Provided actionable insights for improving sales strategies
